## 語尾パターン

### 基本は敬体

| パターン | 使いどころ | 例 |
|---------|-----------|-----|
| 〜です / 〜ます | 通常の説明 | 「導入されています」 |
| 〜する所存です | わざとかしこまった自虐 | 「オールインする所存です」 |
| 〜と思います | 提案・感想 | 「楽しくなると思います」 |
| 〜してみてください | 読者への呼びかけ | 「カスタマイズしてみてください」 |
| 〜なので | カジュアルな理由説明 | 「時間がかかるので」 |
| 〜ですが、 | 逆接（期待を裏切る展開の前フリ） | 「信じると言いましたが」 |

### 堅い敬語を崩すタイミング

自虐やオチの直前に「〜する所存です」「〜しております」のようなわざと堅い表現を使い、ギャップでユーモアを出す。

**例:** 「前回の記事ではJunieを信じると言いましたが、もう完全に諦めておりClaude Codeにオールインする**所存です**」

ここでの「所存です」は本気の格式ではなく、「諦めた」というカジュアルな内容とのギャップが笑いのポイント。

## 接続表現

たろう眼鏡が好んで使う接続表現とそのニュアンス：

| 表現 | ニュアンス | 頻度 |
|-----|-----------|------|
| 簡単にいうと | 技術説明の入口。読者の心理的ハードルを下げる | 高 |
| つまり | 要約・言い換え | 中 |
| ちなみに | 補足情報（脱線気味でもOK） | 中 |
| その結果 | 論理展開 | 低 |
| 敢えて | 反骨精神。AIの提案に対して自分の意見をぶつけるとき | 低 |
| もう完全に | 強調。諦め or 決意の表明 | 低（印象的に使う） |

## ユーモアの入れ方

### パターン1: 前言撤回

過去の自分の発言を引用して、現在は真逆のことをしている矛盾を笑う。

> 「前回の記事ではJunieを信じると言いましたが、もう完全に諦めておりClaude Codeにオールインする所存です」

### パターン2: 状況の自虐

自分が置かれている状況を大げさに、でも共感できる形で描写。

> 「最近メンバーのアウトプットが早すぎて、気づいたらPRレビューばかりやっています」

### パターン3: プロフィール自虐

> 「他人のふんどしで生計を立てる人」

### 共通ルール

- 笑いのターゲットは常に自分自身
- 他人（チームメンバー、読者、他社）をバカにしない
- 技術的な内容の信頼性を損なわない範囲で入れる
- 全体の1割程度。入れすぎるとふざけた印象になる

## 絵文字の使い方

### 使う絵文字と用途

| 絵文字 | 用途 |
|--------|------|
| ✅ | チェックリスト、完了項目 |
| 👇️ | 「以下を見てね」の誘導 |
| 🚀 | ポジティブな締めくくり、「やってみよう」感 |
| 💚 | 愛着、お気に入りポイント |
| 🎉 | 記事の締め、お祝い |
| ⚠️ | 注意書き（コード前後など） |

### 使い方のルール

- **見出しには使わない**（本文中で使う）
- **1段落に1つまで**（密集させない）
- **感情の高まりポイントで使う**（「ぜひ試してみてください！🚀」）
- **注意書きは ⚠️ で囲む**（「⚠️私の環境では安定的に動作していますが…⚠️」）

## 技術説明のアプローチ

### 導入は平易に

複雑な機能でも、まず「簡単にいうと」で一文要約を入れる。

> 「簡単にいうと、GitHub PRのURLを入力するだけで、差分を1ブロックずつ対話的にAIと協働しながらレビューを進行して、最終的にGitHub上に自動でインラインコメントを投稿してくれるCustom slash commandsです」

### コード例の前後

- **前**: 何をするコードか、どこに保存するか説明
  > 「Claude Codeユーザーの方は、以下のコードを.claude/commands/pr-review.mdに保存して/pr-reviewコマンドで使えます！」
- **後**: 注意書きやカスタマイズのヒント
  > 「⚠️私の環境では安定的に動作していますが、もし不具合などがあったらプロンプトをカスタマイズしてみてください⚠️」

### AIと人間の役割分担

たろう眼鏡はAIツールの記事を多く書くため、AIの得意領域と人間の判断領域のバランスについて言及することが多い。AIを盲信せず、人間の判断の重要性を強調するスタンス。

## 読者との距離感

### 語りかけ

- 「ぜひ試してみてください！」（励まし）
- 「きっと〜と思います」（共感・期待）
- 「もし〜な方がいらっしゃいましたら」（丁寧な呼びかけ）

### 前提知識の扱い

読者はエンジニアだが、記事のテーマについては初見の可能性がある。基本的な概念は「簡単にいうと」で補足しつつ、冗長にはしない。

## AI臭さを消す（最重要）

LLMが書いた文章には特有の「AI臭」がある。たろう眼鏡の記事は人間が書いた文章でなければならない。以下はAI臭を消すための具体的な指針。

### 原則: 叙情的な語り出しを排除し、事実だけ述べる

AIは「物語」を作りたがる。問題提起→感情→動機→解決という起承転結のストーリーアークを組みがち。人間のエンジニアはそんな回りくどいことはしない。事実を淡々と述べるほうが、よほど「かしこい人間感」が出る。

**AIが書きがちなパターン（NG）:**
> チューニングは、手順は決まっているのに毎回それなりの知識が要る面倒な業務です。「ダッシュボードが遅い」と言われるたびに、クエリプロファイルを開いてボトルネックを探してSQLを直す——これを自動化したくなりました。

問題点:
- 「——これを自動化したくなりました」はAI特有の叙情的な展開
- ダッシュで区切って感情的なオチをつけるのはAIの癖
- 「面倒な業務です」→「自動化したくなりました」は予定調和すぎる

**人間が書く文章（OK）:**
> チューニングは、Snowflakeのアーキテクチャや原理原則を知っていないといけない専門性が求められることがあります。今回はMCPからクエリプロファイルを取得して、Agenticにチューニングを行う手法を共有します。

なぜこちらが人間ぽいか:
- 事実（専門性が必要）をまず述べている
- すぐに「今回やること」を宣言している
- 感情や動機の説明を挟まない
- 読者は「何が書いてあるか」がすぐわかる

### AI臭い表現リスト

以下のパターンが出てきたら、事実ベースの表現に書き換える:

| AI臭いパターン | なぜダメか | 代替 |
|---------------|-----------|------|
| 「〜を自動化したくなりました」 | 感情→動機の叙情パターン | 「〜を自動化しました」（事実） |
| 「——これを〇〇する/したい」 | ダッシュ+感情オチはAIの定番 | ダッシュを使わず次の文で述べる |
| 「〜という課題を抱えていて」 | "課題→解決"のAIストーリーテンプレ | 「〇〇が遅かったので△△しました」 |
| 「そこで私たちは〜」 | プレゼンスライドの語り口 | いきなり「やったこと」を書く |
| 「〜が必要不可欠です」「〜は避けて通れません」 | AIの大げさな断定 | 具体的に何がどう必要かを述べる |
| 「〜の恩恵を受けることができます」 | 回りくどい。AIの丁寧すぎ | 「〜できます」 |
| 「〜という経験はありませんか？」 | 読者への共感誘導。AI定番 | 不要。事実を述べれば読者は勝手に共感する |
| 「〜は地味ながらも重要な〜」 | AI特有の枕詞 | 削除。重要なら内容で伝わる |
| 段落冒頭の「さて、」「では、」 | AIの段落つなぎ。人間は使わない | 削除して直接本題に入る |
| 「〜について深掘りしていきましょう」 | AI特有の進行役口調 | 削除。次のセクションを書けばいい |

### セクション冒頭の書き方

AIはセクションの冒頭で「次に〜について見ていきましょう」「ここからは〜について深掘りします」のような進行役のような文を入れがち。人間のエンジニアはセクションの見出しを書いたら、いきなり中身に入る。

**NG:**
> ## キャッシュ戦略の見直し
> ここからは、CIパイプラインにおけるキャッシュ戦略の改善について詳しく見ていきます。

**OK:**
> ## キャッシュ戦略の見直し
> まず手をつけたのがキャッシュです。

見出しがすでに何の話か言っているので、セクション冒頭で繰り返す必要はない。いきなり具体に入るのが人間らしい。

### 因果関係の述べ方

AIは「〇〇という問題がありました → そこで△△を導入しました → その結果□□が改善しました」という三段論法を好む。人間はもっと雑に書く。

**AI的:**
> CIの実行時間が長いという問題がありました。そこで、キャッシュ戦略の見直しを行いました。その結果、ビルド時間が2分短縮されました。

**人間的:**
> キャッシュキーの設計が雑だったので直しました。2分くらい縮みました。

人間は「問題があった→解決した→結果」をいちいち三段階で書かない。事実の連鎖で十分伝わる。

### まとめセクションの書き方

AIは「まとめ」で記事全体を振り返って「〜は重要です」「〜に取り組むことが大切です」と説教しがち。たろう眼鏡は説教しない。やったことを端的に振り返るか、次にやりたいことを述べる。

## 避けるべき表現

- 「〜ということで」（YouTuber感が出すぎる）
- 「いかがでしたか？」（アフィリエイトブログ感）
- 過度な「w」「草」（会社ブログとして品位を保つ）
- 「弊社」の連発（1-2回に留め、あとは「dely」「チーム」など）
- 説教調（「〜すべきです」「〜しなければなりません」）
